# SuperPowers 基础入门

## 1. 什么是 SuperPowers？

SuperPowers 是为 Claude Code（及其他 AI 编程助手）设计的**开发工作流插件**。它不是一个代码库或框架，而是一套**结构化的开发流程规范**，通过 14 个 Skill（技能）覆盖软件开发的完整生命周期：

```
构思 → 设计 → 计划 → 实现 → 测试 → 调试 → 审查 → 完成
```

简单来说：SuperPowers 让 AI 助手按照专业软件工程师的最佳实践来工作，而不是随意写代码。

## 2. 核心理念

SuperPowers 建立在四个基本原则之上：

### 2.1 测试驱动开发 (TDD)

```
核心铁律：没有失败的测试，就没有生产代码
```

- 先写测试，看它失败
- 写最少的代码让测试通过
- 重构，保持测试通过
- 循环往复

### 2.2 系统化而非临时性

```
核心铁律：没有根因分析，就没有修复
```

- 遇到 bug 不要猜测，要系统调查
- 每个项目都经过设计阶段，无论多"简单"
- 流程优于直觉

### 2.3 证据优于声明

```
核心铁律：没有验证证据，就没有完成声明
```

- 说"测试通过了"之前，必须实际运行测试
- 说"bug 修好了"之前，必须看到验证结果
- 不接受"应该没问题"

### 2.4 复杂性最小化

```
原则：YAGNI (You Ain't Gonna Need It)
```

- 不添加不需要的功能
- 不做过度设计
- 最简单的方案就是最好的方案

## 3. 安装与启用

### 3.1 安装

在 Claude Code 中运行：

```
/plugin superpowers
```

安装后需要重新加载：

```
/reload-plugins
```

### 3.2 验证安装

安装成功后，插件文件位于：

```
C:\Users\CM640\.claude\plugins\cache\claude-plugins-official\superpowers\5.0.7\
```

### 3.3 支持平台

| 平台 | 安装方式 | 状态 |

## 5. Skill（技能）一览

### 按开发阶段分类

#### 阶段一：构思与设计

| Skill | 名称 | 说明 |
|-------|------|------|
| `brainstorming` | 头脑风暴 | 将想法转化为完整设计文档 |

#### 阶段二：规划

| Skill | 名称 | 说明 |
|-------|------|------|
| `writing-plans` | 编写计划 | 将设计转化为分步实施计划 |

#### 阶段三：实施

| Skill | 名称 | 说明 |
|-------|------|------|
| `subagent-driven-development` | 子代理驱动开发 | 用独立子代理逐任务执行计划 |
| `executing-plans` | 执行计划 | 在独立会话中执行计划 |
| `using-git-worktrees` | Git Worktree | 创建隔离开发环境 |
| `test-driven-development` | 测试驱动开发 | RED-GREEN-REFACTOR 循环 |

#### 阶段四：调试与验证

| Skill | 名称 | 说明 |
|-------|------|------|
| `systematic-debugging` | 系统化调试 | 四阶段根因调查流程 |
| `verification-before-completion` | 完成前验证 | 必须提供证据才能声称完成 |

#### 阶段五：审查与协作

| Skill | 名称 | 说明 |
|-------|------|------|
| `requesting-code-review` | 请求代码审查 | 使用专业代理进行审查 |
| `receiving-code-review` | 接收审查反馈 | 技术评估审查建议 |
| `dispatching-parallel-agents` | 并行代理调度 | 并行处理多个独立问题 |

#### 阶段六：收尾

| Skill | 名称 | 说明 |
|-------|------|------|
| `finishing-a-development-branch` | 完成开发分支 | 合并/PR/保留/丢弃 |

#### 元技能

| Skill | 名称 | 说明 |
|-------|------|------|
| `using-superpowers` | 使用 SuperPowers | 系统使用指南 |
| `writing-skills` | 编写 Skill | 创建新的自定义 Skill |

## 6. 如何调用 Skill

在 Claude Code 中，Skill 通过 `Skill` 工具自动调用。你不需要手动输入命令，Claude 会根据你的需求自动选择合适的 Skill。

### 自动触发

当你描述任务时，Claude 会自动判断并使用相关 Skill：

| 你说的话 | Claude 使用的 Skill |
|----------|-------------------|
| "我想做一个新功能..." | `brainstorming` |
| "帮我修这个 bug" | `systematic-debugging` |
| "写一个用户登录功能" | `brainstorming` → `writing-plans` → ... |
| "执行这个计划" | `subagent-driven-development` 或 `executing-plans` |

### 手动指定

你也可以直接告诉 Claude 使用特定 Skill：

```
请使用 brainstorming skill 来讨论这个功能
请用 TDD 的方式来实现
用系统化调试方法来查找这个 bug
```

## 7. 理解 Skill 的优先级

```
用户指令 (CLAUDE.md / 直接要求)  →  最高优先级
        ↓
SuperPowers Skill 规则           →  覆盖默认行为
        ↓
系统默认行为                     →  最低优先级
```

**重要：** 如果你的 CLAUDE.md 里写了"不要使用 TDD"，那么 TDD Skill 的规则会被覆盖。用户指令永远优先。

## 8. Skill 的两种类型

### 刚性 Skill（必须严格遵循）

- `test-driven-development` — TDD 铁律不可违反
- `systematic-debugging` — 调试流程不可跳过
- `verification-before-completion` — 验证不可省略

### 柔性 Skill（原则可灵活适配）

- `brainstorming` — 设计过程可根据项目复杂度调整
- `writing-plans` — 计划粒度可根据任务规模调整
